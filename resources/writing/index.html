Writing

Make it a Habit

  • Write Daily. Even if just for 15 minutes. Regular short intervals are much more productive than irregular long intervals.
  • Start writing from the beginning of your project. Get in the habit of writing.
  • Create an environment where you can focus (see distractions).
  • If you have a hard time getting into the habit, record the time you spent writing and share it with someone daily.
    When performance is measured, performance improves. When performance is measured and reported, the rate of improvement accelerates.
    -- Thomas S. Monson

Writing Tips

  • I like to start by editing what I wrote the previous day (about 5 mins), then continue with new writing from there. This help me get started, which is often the hardest part, and quickly gets me reoriented to start writing new content.
  • When starting a new paper it can be helpful to begin in bullet form. Slowly add, reorder, and rephrase the bullets until you have a complete outline. Once you have worked out the logical flow with well-written topic sentences, filling in the details is easy.
  • When writing new paragraphs just put things down on paper. Don't try to make it perfect. This is not the time for editing, or jumping over to the web to get references. If you know you need a references, just make a note of it and keep writing.
  • Keep in mind the audience you are writing for.

Editing

  • Start getting feedback early from experts and non-experts. I have found it effective to create a writing group with peers. In a writing group you might meet once a week and each share and critique a paragraph or two that you wrote that week.
  • When you receive feedback, just listen. Don't try to explain what you meant, just listen. If they didn't understand it, then it wasn't clear.
  • Take advantage of BYU's Writing Center.
  • Read your text out loud to yourself.

Logistics

  • Use LaTeX.
  • Use vector graphics whenever possible (pdf, eps). All plots that you create in Python, Matlab, etc. can be output in a vector format. If you have an image that can not be vector-based, like a CFD screenshot, use a high-resolution bitmap.
  • I mentioned this already in research, but be sure to save the data that produced your figures. You will almost always need to change them later.
  • Captions should be full, descriptive sentences.
  • Use hyperref in latex so that your document is properly hyperlinked
  • Be sure to embed fonts before submitting your paper for publication. pdflatex does not do this for you. An easy way to do this on OS X is to open up the PDF then click on File > Print. In the print dialog click on the PDF dropdown menu and select "Save as PDF...". The resulting copy will have all of its fonts embedded. To be clear, you only need to do this on the final version you submit for publication (particularly for a conference, journals will typeset your paper themselves).
  • I recommend using version control for your papers. Use an appropriate .gitignore file (see here for a good start) so that you only commit the source files and not intermediate build files. Provide clear comments or tags on the important milestones (e.g., initial submission to journal) so that it will be easy to compare changes later. You will need to detail changes in your response to reviewer comments. latexdiff is a useful tools for visualizing differences between LaTeX documents.
  • Avoid using rainbow colorbars for contour plots. The ordering of the colors is nonintuitive and does not vary continuously in luminance. The result is an obscuring of details and the perception of artificial boundaries between hues.

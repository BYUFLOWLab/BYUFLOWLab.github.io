Prospective Students

You are welcome to attend our group meetings to learn more about what we are working on (currently Thursday at 10AM in 431A CTB). If you are interested in contributing to an existing project or have an idea that you are interested in exploring, send me your resume and a one page statement of interest. Most of our projects involve scientific computing so it would be helpful to be comfortable with a programming language. Any language is fine but we primarily use the following (in order of frequency of use): Python (v2), Matlab, Julia, Fortran, C, C++. We use Git for version control. A useful resource to learn more about High Performance Scientific Computing is available here: Coursera course, course notes. If you are interested in CFD, you should work through some of the StarCCM+ tutorials that are available in the CAEDM lab.

Beginning Researchers

Your goal as a researcher is to make an impact for good in the world and improve the way we design wind and flight systems. The primary way to make an impact is through tackling challenging and important questions, performing high-quality, insightful work, collaborating with industry and other researchers, and sharing findings broadly at conferences and in peer-reviewed journals.

Open Source Codes

  • Akima a 1D Akima spline interpolation method with analytic derivatives (Python/Fortran)
  • Tools we developed for NREL WISDEM, which is a larger set of models designed for assessing wind plant cost of energy. All of these are written in Python (some are Fortran, C, or C++ wrapped in Python) and implemented in the OpenMDAO framework.
    • CCBlade: a blade-element momentum method with guaranteed convergence behavior (beneficial for gradient-based optimization).
    • pBEAM: a beam finite element method.
    • AirfoilPreppy: a set of utilities for preprocessing airfoil aerodynamic data.
    • pyFrame3DD: a wrapper to an existing finite element code with a few custom additions.
    • RotorSE: a wind turbine rotor analysis module coupleing CCBlade, pBEAM, PreComp and other methods for rotor aero/structural analysis.
    • TowerSE: a wind turbine tower analysis module using pBEAM, cylinder drag theory, linear wave theory, and methods from Eurocode and GL.
    • TurbineSE: a wind turbine module integrating RotorSE, NacelleSE, and TowerSE
    • LandBOS: a C/Python implementation of a balance-of-station cost model.
    • CommonSE: a collection of common utilities shared amongst the WISDEM modules.
    • FLORISSE, a HAWT wake code with analytic gradients

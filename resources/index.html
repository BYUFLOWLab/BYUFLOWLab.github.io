Prospective Undergraduate Researchers

How to get involved

ME 273 is a prerequisite for all of our projects. Some projects might require additional prerequisites like ME 312.

There are three ways to get involved: volunteer, ME 497R, or paid position. We generally discourage the volunteering route as it is difficult to maintain with a busy undergraduate schedule. 497R is usually the ideal route for new researchers. Registering for ME 497R opens up time in your schedule for research, and the course fulfills a tech elective towards graduation. Registering for 3 credits requires about 9-10 hours of work per week (2 credits for 6 hours is also possible). The course is graded, and you will be expected to produce an end-of-semester report. Those who have completed a 497R in our lab are generally our first picks for paid positions.

Those with more experience (either through other research labs, relevant internships, or coursework) may choose to apply directly to a paid position. Most paid positions require a minimum one-year commitment (we also commit to you). Our goal is to help each undergraduate student to contribute to impactful research publications. It takes a semester or two just to get up to speed, and so short-term research commitments are significantly less effective. This commitment is another reason why we recommend completing a 497R first. The 497R project does not require any long-term commitment and gives both you and us a chance to see if a longer term role is a good fit.

Project Description: The NOAA and ESRL have compiled Radiosonde Data into a database to be used for research with the disclaimer that the data is “recent, research-quality data that has not been subjected to production quality control procedures.” The data is available for request in a raw FSL (ASCII text) format in batches of 25,000 soundings. The project is to:

  1. Request batches of sounding data at applicable bands of latitude and longitude at various times and dates.
  2. Write a script to read in the files and parse the data for valid soundings at altitudes relevant to the HALE aircraft research project.
  3. Appropriately average and plot the wind vs. altitude data.
  4. Find an appropriate regression model for the data to be used in simulation.
  5. Similarly, create a model for 99th percentile winds at relevant altitudes.

Time Expected: 5-10 hours depending on programming ability.

Prerequisites: ME 273

Resources:

  • Data Request Page: http://esrl.noaa.gov/raobs/
  • FSL Rawinsonde date format key: http://esrl.noaa.gov/raobs/intl/fsl_format-new.cgi

Project Description: Our lab is currently developing a reduced-order wake model for vertical-axis wind turbines (VAWTs) that will allow an optimizer to position VAWTs in a wind farm to minimizes wake interference. We have currently developed an initial wake model based on computational fluid dynamic (CFD) data from several VAWT simulations that is capable of predicting with reasonable accuracy the wake behavior of a single VAWT. We are looking to expand this wake model to include multiple wind turbines and their interacting wakes, exploring effects of various blade quantities and shapes, and better understand the three-dimensional nature of VAWT wakes. For this project, we are look for an individual to complete the following tasks:

  1. Learn how to use and becoming familiar with Star-CCM+
  2. Work with lower fidelity models to gain a basic understanding of wake interaction and guide decisions on where to focus CFD simulations.
  3. Obtain additional VAWT wake data of single and multiple turbines

Time Expected: 5-10 hours

Prerequisites: ME EN 312. Prior experience with Star-CCM+ preferred. Prior or current enrollment in ME EN 412 preferred.

Project Description: Sandia has developed a simulation code called CACTUS (Code for Axial and Cross-flow TUrbine Simulation) which allows a user to model turbines and obtain information about aerodynamic forces and wake velocity deficits in a matter of minutes. This allows for optimization of turbines in a reasonable amount of time due to the speed of the code. Currently, Sandia is in need of wake validation of a 34-meter vertical-axis wind turbine (VAWT) using the CACTUS code to ensure that the numerical methods match experimental behavior. This project entails:

  1. Obtaining experimental VAWT data from Sandia documentation.
  2. Running CACTUS with the correct parameters identified to reflect the VAWT geometry and speed.
  3. Comparing the results of the CACTUS code to the Sandia data and report on the findings between the two.

Time Expected: 5-10 hours

Prerequisites: ME EN 273. ME EN 312.

Notes: The use of the CACTUS code could be expanded to explore novel configurations, parked and tilted VAWTs, and modify the code to handle multiple turbines.

Open Source Codes

  • Akima a 1D Akima spline interpolation method with analytic derivatives (Python/Fortran)
  • Tools we developed for NREL WISDEM, which is a larger set of models designed for assessing wind plant cost of energy. All of these are written in Python (some are Fortran, C, or C++ wrapped in Python) and implemented in the OpenMDAO framework.
    • CCBlade: a blade-element momentum method with guaranteed convergence behavior (beneficial for gradient-based optimization).
    • pBEAM: a beam finite element method.
    • AirfoilPreppy: a set of utilities for preprocessing airfoil aerodynamic data.
    • pyFrame3DD: a wrapper to an existing finite element code with a few custom additions.
    • RotorSE: a wind turbine rotor analysis module coupleing CCBlade, pBEAM, PreComp and other methods for rotor aero/structural analysis.
    • TowerSE: a wind turbine tower analysis module using pBEAM, cylinder drag theory, linear wave theory, and methods from Eurocode and GL.
    • TurbineSE: a wind turbine module integrating RotorSE, NacelleSE, and TowerSE
    • LandBOS: a C/Python implementation of a balance-of-station cost model.
    • CommonSE: a collection of common utilities shared amongst the WISDEM modules.
    • FLORISSE, a HAWT wake code with analytic gradients
